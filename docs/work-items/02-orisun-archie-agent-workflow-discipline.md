## Outcome

Capture a separate work item for the workflow-discipline side of agent-assisted
development in Orisun and Archie. This item should describe how an operator
adjusts expectations, slows down when necessary, and spends more time refining
with the coding agent so the overall method becomes dependable rather than
merely fast.

## Decision Changes

This work item locks in that early throughput is not the main success metric
for this mode of work. The operator should expect to invest time in iteration,
constraint tuning, and review before the workflow becomes efficient. It also
locks in that refinement with the agent is part of the primary work, not an
optional cleanup phase after an initial answer.

The item should remain focused on operating discipline and workflow structure.
It should not absorb the broader craft question from the first work item, and
it should not turn into a philosophical discussion about ownership or
existential responsibility.

## Main Quests

- Define the recommended interaction loop for working with a coding agent when
  quality and controllability matter more than raw speed.
- Describe how an operator should decide when to keep refining an answer versus
  when to reset, narrow scope, or change the prompt structure.
- Identify the repository scaffolding, standards, and supporting documents that
  make this slower but more reliable workflow viable.
- Compare where Orisun and Archie need shared guidance versus project-specific
  operating rules.

## Acceptance Criteria

The work item is complete when it produces guidance that can be translated into
practical operating documentation for both repositories. That guidance must
make expectation-setting explicit, define the refinement loop clearly, and show
how the workflow becomes more dependable over time.
