## 1. Follow PEP 8 (The Style Guide)

* Naming Conventions: Use `snake_case` for variables and functions,
  `PascalCase` for classes, and `UPPER_SNAKE_CASE` for constants.
* Indentation: Use 4 spaces per indentation level.
* Line Length: Limit all lines to a maximum of 79 characters.
* Imports: Keep imports at the top of the file.

## 3. Use Type Hinting

While Python is dynamically typed, introducing type hints drastically improves
code clarity.

## 5. Document Effectively

* Docstrings: Write descriptive docstrings for all public modules, functions,
  classes, and methods.

## Personal preferences

Less is more: a cleaner, simpler approach over lots of features and "production
ready" code.
